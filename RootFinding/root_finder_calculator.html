Root-Finder Calculator

Below, you may provide a single-variable function (in JavaScript syntax) to find a root of. If the function has named constants, a value must be assigned to the named constant whether it be a pure number or a JavaScript expression in terms of a constant above it. You may add/remove constants below the function text input.

For example, it could be the following function if the square root of a number a is desired:

return x ** 2 - a;

Then, below, you would add a constant a and assign it a value, say, 2, if you wanted to approximate the square root of two.

You may add layers of root-finding methods below the constants. Each layer contains a root-finding algorithm, an error tolerance, and, optionally, a max iteration count. If no max iteration count is supplied, then the method will never terminate.

Note that when adding layers, the choices are limited to the following rules, or the algorithm will halt:

  • If previous root-finder was either bisection or regula-falsi, then the current layer can be any choice.
  • If previous root-finder was the secant method, then neither bisection nor regula-falsi can be selected for the current layer.
  • If previous root-finder was either Newton's method or modified Newton's method, then only either Newton's method or modified Newton's method may be chosen for current layer.

For example, if you would like to start with bisection with an intermediary error tolerance of 1E-2, then switch to regula-falsi with an intermediary error tolerance of 1E-5, then switch to Newton's method with a final error tolerance of 5E-14, then you would do the following:

  • Add layer with bisection method at an error tolerance of 1E-2 and an optional max iteration count.
  • Add layer with regula-falsi at an error tolerance of 1E-5 and an optional max iteration count.
  • Add layer with Newton's method at an error tolerance of 5E-14 and an optional max iteration count.
